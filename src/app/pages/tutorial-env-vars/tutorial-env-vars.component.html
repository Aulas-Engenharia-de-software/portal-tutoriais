Ocultando Credenciais da AWS

Subir suas chaves da AWS (`Access Key` e `Secret Key`) no GitHub é uma vulnerabilidade crítica. Bots escaneiam repositórios públicos em segundos para sequestrar contas.

Sem contar que a própria AWS ao detectar sua chave em algum servidor do github, bloqueia a sua chave no mesmo instante.

Vamos aprender a esconder essas chaves usando Variáveis de Ambiente na sua IDE.

1

Ajuste no application.properties

Em vez de colocar as chaves reais diretamente no arquivo, nós informamos ao Spring Boot para buscar essas informações do sistema operacional no momento da execução.

Atenção: Apague suas chaves reais do código agora mesmo! 


# Credenciais AWS Dinâmicas
spring.cloud.aws.credentials.access-key=${AWS_ACCESS_KEY_ID}
spring.cloud.aws.credentials.secret-key=${AWS_SECRET_ACCESS_KEY}
spring.cloud.aws.region.static=us-east-1

# Filas
queue.order-events=T0XN_SEU_NOME_PEDIDOS.fifo

A sintaxe ${VARIAVEL} diz ao Spring para injetar o valor automaticamente.

2

Configurando no IntelliJ IDEA

Se você usa o IntelliJ, as variáveis são injetadas na sua configuração de execução (Run/Debug Configurations).

A

No canto superior direito, ao lado do botão de "Play", clique na seta para baixo na sua aplicação e selecione Edit Configurations...

B

No campo Environment variables, adicione suas chaves no formato CHAVE=VALOR separadas por ponto e vírgula (;), conforme abaixo:

AWS_ACCESS_KEY_ID=sua_access_key;AWS_SECRET_ACCESS_KEY=sua_secret_key
C

Caso não apareça o campo de Environment Variables para você, clique em Modify Options e habilite a configuração, conforme abaixo:

D

Ao final, sua configuração de execução deve ficar conforme a imagem abaixo, com as variáveis de ambiente devidamente configuradas:

E

Clique em Apply e depois em OK para salvar as configurações e pronto!
Agora, quando você executar sua aplicação usando essa configuração, as chaves da AWS serão injetadas automaticamente, mantendo seu código seguro!

3

Configurando no VS Code

Se você usa o Visual Studio Code com a extensão Java, as configurações ficam no arquivo launch.json.

A

Abra a barra lateral de execução (Run and Debug) e clique na engrenagem para abrir o launch.json.

B

Caso a tela acima não apareça, provavelmente irá aparecer uma opção para voce criar o arquvo launch.json, conforme abaixo:

Clique em create a lauch.json file, conforme indicado na imagem:

C

Caso voce tenha as extensões do Java instaladas, provavelmente irá aparecer a opção de configuração para Java, escolha ela, conforme abaixo:

D

Dentro da sua configuração atual, adicione a propriedade "env" com as suas chaves reais, conforme o exemplo abaixo:

"configurations": [
  {
    "type": "java",
    "name": "ArquiteturaHexagonalApplication",
    "request": "launch",
    "mainClass": "com.fag.lucasmartins.arquitetura_software.ArquiteturaHexagonalApplication",
    "projectName": "t02n-arquitetura-software",
    "env": {
        "AWS_ACCESS_KEY_ID": "COLE_SUA_ACCESS_KEY_AQUI",
        "AWS_SECRET_ACCESS_KEY": "COLE_SUA_SECRET_KEY_AQUI"
    }
  }
]
E

Ao final da sua configuração das variaveis de ambiente, seu arquivo launch.json, assim como a estrutura de pastas deve ficar conforme o seguinte:

Aplicação Blindada!

Você agora pode commitar e enviar seu código para o GitHub tranquilamente. O código está seguro e as chaves existem apenas na sua máquina local!
