Laboratório 01

Manual de Integração de Eventos na AWS

Este tutorial guia você na configuração de uma arquitetura orientada a eventos (EDA) utilizando IAM para segurança, SQS FIFO para mensageria e SNS para distribuição de notificações.

O líder do grupo não deve compartilhar sua senha "Root". Criaremos um usuário IAM específico para o time.

Pesquise por IAM na barra de pesquisa do console da AWS e vá em Usuários(Users). Você deve encontrar uma tela conforme a seguinte: Clique em Criar usuário(Create User). Nome do usuário: Grupo_Arquitetura_Software_AWS Escolha a opção Anexar politicas diretamente (Attach policies directly) Pesquise na caixa de texto e anexe as seguintes políticas:
  • AmazonSQSFullAccess
  • AmazonSNSReadOnlyAccess
No último passo da criação, você deverá ver uma tela conforme imagem abaixo:
Acesse o usuário que voce acabou de criar. Grupo_Arquitetura_Software_AWS.
Clique em Criar chave de acesso (Create access key).
Escolha a opção "Código Local".
Na opção de Definir etiqueta de descrição, dê uma etiqueta como Chave para consumo de eventos SQS .
Faça o Download do arquivo .csv com as suas credenciais criadas

MUITO IMPORTANTE: Baixe o arquivo .csv. A "Secret Key" só é exibida uma vez!
