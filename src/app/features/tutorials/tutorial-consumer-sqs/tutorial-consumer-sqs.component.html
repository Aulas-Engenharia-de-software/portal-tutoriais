Consumidor SQS

Configurando o Consumer SQS

Agora que sua infraestrutura na AWS está pronta, vamos conectar sua aplicação Java/Spring ao barramento de eventos. Seguiremos o padrão de Adapters para isolar a tecnologia da fila da nossa regra de negócio.

Dependências do Projeto


Certifique-se de que o seu pom.xml possui o starter da AWS Cloud. Ele contém as bibliotecas necessárias para o @SqsListener.
Adicione tambem o gerenciador de dependências abaixo para gerenciar as versões da SDK da AWS.

Configuração de Credenciais


No arquivo src/main/resources/application.properties, adicione as chaves que você baixou no arquivo .csv da Fase 1.

Segurança

Nunca suba suas chaves reais para o GitHub. Em produção, use Variáveis de Ambiente!

Criando o SQS Adapter


Esta classe será responsável por "ouvir" a fila. Observe o uso da DTO para receber dados de uma fonte externa.

Dica de Sucesso

Certifique-se de que sua classe DTO tenha os mesmos nomes de atributos do JSON enviado pelo professor, ou use a anotação JsonProperty do Jackson para mapear os nomes.

Pronto para Testar!

Rode sua aplicação e aguarde o log "Mensagem recebida". Se você configurou tudo certo, o pedido aparecerá no banco de dados em segundos.
